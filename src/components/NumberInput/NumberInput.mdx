# \<NumberInput />

NumberInput is a special input to handle number input.

### Use in `react-hook-form`

`onChange` in `<NumberInput />` will return formatted `string` value, \
so using `register()` is not recommended in this component.

```tsx
import { NumberInput } from "@reactleaf/input"

function FormComponent() {
  const { control } = useFormContext()

  return (
    <Controller
      name="withFormatter"
      control={control}
      render={({ field, fieldState }) => (
        <NumberInput
          label="WithFormatter"
          min={-999_999_999}
          max={999_999_999}
          formatter={customFormat.formatter}
          parser={customFormat.parser}
          onValueChange={field.onChange} // <- This is important
          onFocus={field.onFocus}
          onBlur={field.onBlur}
          errorMessage={fieldState.error.message}
        />
      )}
    />
  )
}
```

## Props

`<NumberInput />` extends almost all props of `<input />` like `<TextInput />`, but excepts, `type`.

> **WARNING** \
> `@reactleaf/input` extends basic `<input />`, so `onChange` recieves `event` as a parameter. \
> Reading `event.currentTarget.value` from event listeners would be a text value, not a number. \
> If you want to get numeric value, use `onValueChange` instead.
